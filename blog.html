Discuss the scope of var, let, and const

var: Variables declared as var are in the global or function scope. Variables declared outside functions are called global scope, on the other hand, variables declared inside a function are called local variables and in the function scope. It can be redeclared and reassigned.
let: Variables declared with let are in the block scope. Variables declared in a block are called block scope. It can be reassigned to other values, but they cannot be redeclared.
const: Variables declared as const are block scope. It is used for declaring constant variables that cannot be changed. They cannot be redeclared, and reassigned.

In summary, the scope of var is function scoped, let and const are block-scoped. Developers do not recommend use var anymore. They use let or const instead of var. If we are not to change the value of a variable, it is best practice to use const.

Tell us the use cases of null and undefined

Null is a primitive type in JavaScript. It is a special value that represents an empty or unknown value. It is assigned by the programmer that represent intentional absence of value when needed. It can be returned by functions to indicate no meaningful value.

Undefined is a primitive type in JavaScript. Undefined represents a variable that has not been assigned a value, or not declared at all. JavaScript normally assigns it automatically, when a value is missing or not initialized.

What do you mean by REST API?

An Application Programming Interface (API) is a set of rules two software programs use to interact with each other and exchange data. REST stands for Representational State Transfer. A REST API is an API that uses HTTP requests to access and use data. It is also known as RESTful API. REST API are the most commonly used JSON and XML data formats for exchanging data.

A REST API uses HTTP methods to perform CRUD (Create, Read, Update, Delete) operations on resources. For example:
GET: Retrieve data about API resource.
POST: Create an API resource.
PUT: Update an API resource.
DELETE: Remove an API resource.

The key principles of a REST API:

  • Uniform Interface
  • Stateless
  • Client-Server Architecture
  • Cacheable
  • Layered System
  • Code on Demand
